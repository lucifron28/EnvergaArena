# Backend URLs (Simplified)

This file explains what happens in `backend/backend/urls.py` in simple terms.

## Quick idea

The URL config has three main parts:

1. Direct routes (written manually with `path(...)`)
2. Public API routes generated by `router`
3. Admin API routes generated by `admin_router`

`DefaultRouter` automatically creates common REST endpoints for each registered ViewSet.

## Direct routes

These are explicit paths:

- `/admin/` -> Django admin site
- `/api/auth/login/` -> login and issue tokens
- `/api/auth/refresh/` -> refresh token
- `/api/auth/logout/` -> logout
- `/api/auth/me/` -> current authenticated user
- `/api/public/rooney/query/` -> Rooney public query endpoint
- `/api/public/tryouts/send-otp/` -> send OTP for tryout
- `/api/public/tryouts/verify-otp/` -> verify OTP
- `/api/public/tryouts/apply/` -> submit tryout application

## Router-generated public endpoints

All routes below are under `/api/public/`.

### Core

- `departments`
- `venues`
- `venue-areas`
- `news`

### Events

- `events`
- `event-categories`

### Tournaments

- `athletes`
- `tryout-applications`
- `registrations`
- `schedules`
- `match-results`
- `podium-results`
- `medal-records`
- `medal-tally`
- `rooney-logs`

For each ViewSet, DRF usually provides patterns like:

- `GET /api/public/<resource>/` (list)
- `POST /api/public/<resource>/` (create)
- `GET /api/public/<resource>/<id>/` (retrieve)
- `PUT/PATCH /api/public/<resource>/<id>/` (update)
- `DELETE /api/public/<resource>/<id>/` (delete)

(Actual allowed methods depend on each ViewSet's permissions and implementation.)

## Router-generated admin endpoints

All routes below are under `/api/admin/`.

- `news`
- `ai-recaps`

These use separate admin-focused ViewSets.

## Why two routers?

- `router` keeps public/business API routes under `/api/public/`
- `admin_router` keeps admin-only API routes under `/api/admin/`

This makes access control and API organization easier to maintain.

## URL flow summary

- Manual auth + special endpoints are declared directly.
- Most CRUD endpoints are auto-generated through routers.
- Final URL prefixes are attached with:
  - `path('api/public/', include(router.urls))`
  - `path('api/admin/', include(admin_router.urls))`
